IIS Basics


IIS Reset
Restarting or stopping IIS, or rebooting your Web server, is a severe action. When you restart the Internet service, all sessions connected to your Web server (including Internet, FTP, SMTP, and NNTP) are dropped. Any data held in Web applications is lost. All Internet sites are unavailable until Internet services are restarted.

Stopping and Starting App Pool
By stopping an application pool, you are instructing all IIS worker processes serving this application pool to shut down, and prevent any additional worker processes from being started until the application pool is started again.
Starting an application pool enables IIS worker processes to be created to serve requests in that pool.

App Pool Recycle
Recycling an application pool causes all currently running IIS worker processes in that application pool to be gracefully shutdown, but unlike stopping the pool, new IIS worker processes can be started on demand to handle subsequent requests.

App Pool Recycle Schedule

Step 1: Open IIS
Step 2: In the Connections pane, expand the server name, and then click Application Pools.
Step 3: In the Application Pools pane, select the desired application pool.
Step 4: In the Actions pane, click Recycling...

Recycle App Pool

Step 1: Open IIS
Step 2: In the Connections pane, expand the server name, and then click Application Pools.
Step 3: In the Application Pools pane, select the desired application pool.
Step 4: In the Actions pane, click Recycle

IIS Logs

IIS log files are stored by default in the %SystemDrive%\inetpub\logs\LogFiles folder of your IIS server. Example: c:\inetpub\logs\LogFiles

Identify 64-bit App Pool

Step 1: Open command prompt
Step 2: Go to c:\Windows\System32\inetsrv\
Step 3: Run the commnad appcmd list apppool /enable32BitAppOnWin64:false
